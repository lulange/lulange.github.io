The 3PCL method

3PCL or 3 bottom layer edges, Pairs (corner edge pairs like F2L), Corners of last layer, and Last 5 edges is a speedsolving method that is basically a mix of roux and CFOP so that blockbuilding is avoided.

Pros:

  • Less algs than CFOP
  • Easier first step inspection
  • CLL is a 2x2 speedsolving method as well so you learn two methods in one
  • If transitioning from CFOP you can keep most F2L strategies

Cons:

  • Harder to learn recognition (for CLL)
  • Not used by anyone but me
  • Influencing edges with F2L does not really help as many of the CLL algs will mess with the LL edges
  • Somewhat of a high turn count compared to some speedsolving methods

The steps:

  1. 3 bottom layer edges
  2. Pairs like F2L in CFOP
  3. CLL or Corners of Last Layer
  4. L5E or Last 5 Edges

3 bottom layer edges:

This step should be fairly easy as it is similar to most beginner methods as well as CFOP. All you have to do is solve 3 of the edges of one side (usually the white).
This step can take anywhere from 0-6 turns.

This step should take 3-6 turns

Pairs:

This step is basically F2L from CFOP with one edge out so that some more cases are possible and some shortcuts are possible. F2L can be done using anywhere from 0-41 algorithms.
If you would like to learn more about F2L either look up intuitive F2L on youtube or go here for F2L algs.

This step should take 20-30 turns

CLL:

CLL has 42 cases separated by the corner pattern into 8 groups. For a complete list of the CLL algorithms that I use, click here.

This step averages 11 turns

L5E:

L5E has two substeps. First L5EO or Last 5 Edges Orientation which has only five quick cases. Second L5EP or Last 5 Edges Permutation which has only 16 equally quick cases.

This step averages 15 turns

Conclusion:

The 3PCL method has only 63 algorithms for the last layer and the Pairs step can be done mostly intuitively. The Pairs step and the 3 bottom layer edges step are both hard to have a specific average on, but for me I think it is about 5 for 3BLE and 25-30 for Pairs, giving me an average of 57-62 turns total.

Turns per solve average 55-65 turns
